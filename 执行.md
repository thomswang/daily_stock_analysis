# 历史行情回填（近 6 年）

数据源：`westock quote --date` → 表 `stock_daily_quote`

## 一键启动 6 个终端（推荐）

在 **Git Bash** 或 **Windows Terminal** 里执行一次即可，会自动打开 6 个窗口/标签页（2021–2026 各一个）：

```bash
cd e:/analysis/daily_stock_analysis
bash scripts/run_backfill_6years.sh
```

或在资源管理器中双击：`run_backfill_6years.bat`

可选环境变量：

```bash
export END_2026=2026-07-04    # 2026 结束日
export SLEEP=0.1              # 默认已是 0.1
bash scripts/run_backfill_6years.sh
```

> 优先使用 Windows Terminal（`wt`）开标签页；没有则每个年份新开一个 Git Bash 窗口。

## 环境（手动单年运行时）

```bash
cd e:/analysis/daily_stock_analysis
export WESTOCK_DATA_DIR=e:/analysis/westock-data
export WESTOCK_QUOTE_SLEEP=0.1
```

> 两处限流均为 **0.1s**：`--sleep`（每只/每段后）+ `WESTOCK_QUOTE_SLEEP`（quote 分批间）。代码默认已改为 0.1，不写参数也是 0.1。

## 常用辅助命令

查看进度：

```bash
python backfill_history.py --progress-status --progress data/progress_2021.json
```

中断后续传（原命令重跑即可，或重新执行一键脚本）：

```bash
# 与上面某年的命令完全相同，已完成的股票会自动跳过
```

仅重试失败项：

```bash
python backfill_history.py --all --mode range --start 2021-01-01 --end 2021-12-31 --progress data/progress_2021.json --sleep 0.1 --retry 2 --retry-failed
```

## 说明

- 共 **5207** 只 A 股；上市日晚于 `--start` 的会自动从上市日起拉，不会空请求 pre-IPO 区间。
- 台账文件：`data/progress_YYYY.json`，每完成一只写一次，可随时中断。
- 多进程并行写同一 SQLite 时，若出现 `database is locked`，可把 `--sleep` 调到 `0.4`，或稍后再 `--retry-failed`。
- **同一年份只跑一个进程**，不要重复启动（一键脚本每个年份只开一次）。
